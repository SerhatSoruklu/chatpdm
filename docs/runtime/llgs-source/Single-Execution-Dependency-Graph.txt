================================================================
CROSS-PHASE HARD DEPENDENCIES
================================================================

Phase 2 depends on Phase 1 because:
  - you cannot make registry authoritative if output/route truth is still ambiguous

Phase 3 depends on Phase 2 because:
  - invariance requires a stable canonical registry and invariant authority

Phase 4 depends on Phase 3 because:
  - exposure integrity is meaningless if semantic validation is not trusted

Phase 5 depends on Phase 4 because:
  - telemetry must observe stable truth, not unstable UI/backend disagreement

================================================================
VERSIONING RULES YOU MUST LOCK
================================================================

V1. registryVersion increments on any registry truth change
V2. policyVersion increments on any decision-rule change
V3. validatorVersion increments on any invariance check change
V4. exposureVersion increments on any exposure mapping change
V5. deterministicKey must include all relevant version inputs

Recommended deterministicKey basis:
  hash(
    inputHash +
    registryVersion +
    policyVersion +
    validatorVersion +
    exposureVersion
  )
