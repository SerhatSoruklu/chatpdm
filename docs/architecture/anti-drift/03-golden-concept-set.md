# Lock 3 - Golden Concept Set

## Purpose

Keep the whole register system aligned to a small reference standard.

Without reference concepts, new packets may pass mechanically while still feeling off.

## Rule

Maintain a fixed gold-standard concept set that defines the expected register quality for the system.

Current reference set:

- `authority`
- `power`
- `legitimacy`
- `duty`
- `responsibility`

## What It Protects

This lock prevents:

- style drift across newly authored concepts
- acceptable-but-weaker register writing
- validator gaming that passes thresholds without preserving system voice

## How To Use It

When authoring or reviewing a new concept, compare it against the golden set:

- does `simplified` read faster than `standard`
- does `formal` feel tighter than `standard`
- does the prose still feel like it belongs in the same system

## Review Standard

Every new public concept should be reviewed against the golden set before it is treated as publishable.

The golden set is not decorative documentation. It is the local calibration surface for register quality.
