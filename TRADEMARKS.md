## Scope

This policy covers the ChatPDM project name, logos, icons, wordmarks, product identity, branding assets, and any associated trade dress or visual identity where applicable.

## General Rule

You may use the code under the software license, but that alone does not permit use of the ChatPDM brand.

You may refer to ChatPDM factually. You may not imply affiliation, endorsement, authorship, sponsorship, or official status unless you have explicit permission.

## Permitted Uses

Permitted uses include:

- factual references to the project name in discussion, documentation, articles, or comparative analysis
- unmodified redistribution of the repository while preserving attribution and license notices
- nominative use needed to describe compatibility, forks, origin, or source history
- truthful links to the project or references to the original repository

## Prohibited Uses

Prohibited uses include:

- using "ChatPDM" in a way that implies official endorsement, sponsorship, or affiliation
- using ChatPDM logos, icons, or branding to market a fork or derivative as official
- modifying and redistributing the project while presenting it as the official ChatPDM project
- using the name, logo, or visual identity in misleading, deceptive, confusing, or reputation-harming ways
- reusing trade dress or visual identity in a way likely to cause confusion about source or status

## Forks and Derivatives

Forks are allowed under the software license, but branding must be changed if the project is modified and redistributed in a way that could cause confusion.

Modified builds should use clearly differentiated naming and branding when distributed publicly. Attribution to the original source may remain, but the fork must not be presented as the official ChatPDM project.

## Domain Names / Handles / Distribution

Using confusingly similar names in domains, package names, app names, organization names, or social handles is not permitted when it implies official association with ChatPDM.

## Requesting Permission

Any use outside the clearly permitted uses in this policy requires prior written permission from the project owner or maintainer.

The software license governs the code. This trademark policy governs names, logos, branding, and associated identity assets.

If there is any tension between the two, treat them as separate layers of rights: license rights for code use, and trademark or branding rights for identity use.
