# Product Identity

This note captures the durable product direction for ChatPDM.

## Core Identity

ChatPDM is:

- a deterministic meaning system
- a predefined deterministic meaning
- a source-grounded concept-resolution product

ChatPDM is not:

- a chatbot clone
- a freeform AI assistant
- an anti-AI marketing project
- a derivative product

## Positioning Rule

- AI is useful for exploration.
- ChatPDM is for cases where wording, meaning, and consistency matter more.

## Workspace Rule

Canonical workspace root:

- `/home/serhat/code/chatpdm`

Launcher folder on Windows:

- `D:\chatpdm.code`

## Frontend Rule

- Keep ChatPDM visually calm, editorial, and system-focused.
- Use the current Angular landing shell as the design baseline.
- Use unique ChatPDM-specific class names.
- Keep the landing page minimal, calm, credible, and product-like.
- Do not let the UI drift into chatbot bubble patterns or generic AI SaaS fluff.

## Backend Rule

- Keep the backend scaffold simple until explicit product logic is requested.

## Content Rule

- Concepts, aliases, sources, optional perspectives, and status/version metadata must stay clearly separated.
- Perspectives are secondary. Canonical concept meaning stays neutral and source-grounded.

## Reference Rule

- Preserve `chatpdm-reference`.
- Keep `nozick` as the intended folder name.
- Keep the `quran` API note as a real source input, not an empty placeholder.
