Resolution contract

This page defines the stable public result shapes returned by the ChatPDM runtime.

Contract purpose

Structured result states, not freeform responses.

The runtime does not generate responses. It returns structured resolution states with declared meaning and stable boundaries.

This page describes contract-bound output behavior. HTTP transport and endpoint details remain separate on the API surface.

Refusal contract

Explicit refusal is valid output.

Refusal is part of the runtime contract. If a query cannot be resolved within the current system, the runtime returns that limit explicitly rather than approximating beyond scope.
