Version policy

This page defines how runtime behavior is versioned and when a change is allowed to alter output.

Core guarantee

Same declared versions, same runtime outcome.

If query, normalization, matcher logic, concept set, and contract version remain the same, the runtime outcome must remain the same.

Replay / citation rule

A runtime result is identified by its declared tuple.

A result can only be cited or replayed precisely when the identifying tuple remains intact.

Transport and debug metadata sit outside this tuple and do not define output identity.
