Deterministic concept resolution inside a bounded runtime.

ChatPDM validates reasoning like a compiler validates code.

Given an argument, it checks:

  • are all terms defined?
  • does each claim have support?
  • are there contradictions?

Example

Input: “Company policy overrides statutory law in all cases.”

Result: Invalid — contradicts defined legal hierarchy.

If the system cannot map the input to known concepts, it refuses instead of guessing.
Refusal is a correct outcome, not a failure.

Every output is traceable, bounded, and derived only from defined concepts.
No hidden reasoning. No probabilistic guesses.
